# L Message UI/UX改善提案サイト

株式会社ミショナのITエンジニア職種別適性テスト提出用に作成したNext.jsサイトです。

L Messageを実際に操作し、公式サイトやマニュアルで説明されている仕様と、初めて利用したときの実際の体験を比較しました。
その中で見つけた課題をIssueとして整理し、改善案・優先度・想定工数・確認方法をまとめています。

## 制作目的

L Messageは、LINE公式アカウント運用に必要な機能が多く用意されているシステムです。

一方で、初めて使うユーザーにとっては、機能名だけでは何をする画面なのか分かりにくく、目的に合った機能を選びにくい場面がありました。

今回は、Luna Hair Salonという美容室アカウントを想定し、リッチメニュー、予約フォーム、自動応答、店舗紹介、アクセス案内を作成しました。
その操作体験をもとに、L MessageのUI/UX改善案を3点提案しています。

## 改善提案

### Issue 1：目的に合う機能を選びにくい

初めてログインしたユーザーにとって、左メニューに並ぶ機能名だけでは、どの機能をどの順番で使えばよいか分かりにくいと感じました。

改善案として、目的別セットアップナビゲーションを追加し、「美容室の予約導線を作る」などの目的を選ぶと、必要な機能だけがハイライトされる導線を提案しています。

### Issue 2：画面サイズによって文字やボタンが見切れる

フォーム作成画面やリッチメニュー編集画面では、画面サイズによって文字やボタンが見切れ、重要な操作を見落とす可能性があると感じました。

改善案として、レスポンシブ表示の改善と、保存ボタン・次へボタンなどの固定ナビゲーションを提案しています。

### Issue 3：リッチメニュー画像の調整が難しい

リッチメニュー画像は指定サイズで登録する必要がありますが、AI画像や自作画像を使う場合、サイズ調整やトリミングが難しいと感じました。

改善案として、画像自動調整機能と、業種別デザインガイドの追加を提案しています。

* ユーザー名：`missiona`
* パスワード：`kadai`

## noindex設定

検索結果に表示されないように、noindex設定を入れています。

`app/layout.tsx` にて、robots設定を以下のように指定しています。

```ts
robots: {
  index: false,
  follow: false,
}
```

## ローカルで確認する方法

```bash
npm install
npm run dev
```

ブラウザで以下を開きます。

```text
http://localhost:3000
```

Basic認証が表示された場合は、以下を入力します。

```text
ユーザー名：missiona
パスワード：kadai
```

## Vercelで公開する方法

1. GitHubにコードをpushする
2. Vercelで「Add New Project」を選択する
3. GitHubのリポジトリを選択する
4. Deployを実行する
5. 発行されたURLを確認する
6. Basic認証とnoindex設定を確認する
7. 提出フォームにURLを入力する

## 提出前チェック

* VercelのURLを開くとBasic認証が表示される
* `missiona / kadai` でログインできる
* ページ内容が正しく表示される

* 公開前チェックリストが表示されている
